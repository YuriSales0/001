# HostMasters — Contexto do Projecto

SaaS B2B de gestão de propriedades de arrendamento de curta duração na **Costa Tropical, Espanha (Granada)**.
Mercado-alvo: proprietários internacionais (UK, Suécia, Noruega, Holanda, Alemanha) que compram casas como investimento.

## Modelo de Negócio — Platform Marketplace

```
HostMasters (plataforma + Crew própria)
        ↕
Managers (rede de closers/CS — atraem proprietários)
        ↕
Proprietários (clientes finais que pagam)
        ↕
Crew (equipa própria do HM, activada on-demand)
```

**Princípio central**: "O teu Manager muda. Os standards não."  
A qualidade vem da Crew própria HM com checklists, fotos e relatórios — não de uma pessoa específica.

## Stack

- Next.js 14 App Router + TypeScript + Tailwind
- PostgreSQL + Prisma ORM
- NextAuth.js (4 roles: ADMIN / MANAGER / CREW / CLIENT)
- Resend (email) · Stripe (pagamentos) · Vercel (hosting)
- Anthropic Claude (AI chat, pricing engine, scraper, monitor)
- Vitest (testes) · i18n: 8 idiomas (EN, PT, ES, DE, NL, FR, SV, DA)

## Os 4 Roles

### ADMIN (Yuri / HM)
Gestão total da plataforma. Recruta Managers, controla standards da Crew, responsável legal/fiscal.

### MANAGER
- **Papel**: SDR + Closer + Customer Success (não operacional)
- **Perfil**: expatriado já a viver na Costa Tropical, fala nativo a língua dos proprietários
- **Modelo Uber**: qualquer pessoa com perfil certo, sem licença obrigatória
- **Território exclusivo** por zona, contrato mínimo 6 meses

### CREW
- Execução operacional (limpezas, check-in/out, manutenção)
- Própria do HM, paga via plataforma em 24h
- Trabalho year-round graças a manutenções preventivas

### CLIENT (Proprietário)
- Portal de transparência total: earnings, bookings, care, fiscal

## Property Lifecycle

```
PENDING_CLIENT → PENDING_APPROVAL → CONTRACT_PENDING → ACTIVE
                                  └─ Admin aprova   └─ Cliente assina contrato
```

Pré-requisitos para approval: ≥1 foto + house rules seleccionadas.

## House Rules

34 regras pré-definidas em 7 categorias (`src/lib/house-rules.ts`).
**NÃO há texto livre** — proprietário só selecciona checkboxes.

## Regras importantes

- **Moeda**: base EUR. `useCurrency()` é só para display — nunca usar em inputs de valor real (usar `fmtEUR` fixo).
- **subscriptionPlan**: não está na sessão NextAuth — sempre ler da DB com `prisma.user.findUnique`.
- **Comissões**: nunca hardcodar — usar `PLAN_COMMISSION` e `calcCommission` de `src/lib/finance.ts`.
- **Invoices auto-gerados**: `isAutoGenerated = true` nos criados ao pagar payouts — não contam como receita HM.
- **Migrations**: criar ficheiro SQL em `prisma/migrations/` com timestamp, não usar `db push` em produção.
- **House rules**: só keys da constante `HOUSE_RULES`, nunca texto livre.
- **Qualidade**: garantida pelo sistema (checklists + fotos), não pelo Manager.
